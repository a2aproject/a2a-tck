## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
In the compatibility report (`reports/compatibility.html`), test IDs are displayed as `<span>` elements with the full pytest node ID in a tooltip. These test IDs should be clickable hyperlinks that navigate to the corresponding test entry in `reports/tck_report.html`.

**Current state:**
- `compatibility.html` shows test IDs per requirement (e.g., `TestAgentCardDiscovery::test_agent_card_retrievable`) as plain text spans with the full node ID in the `title` attribute
- `tck_report.html` (generated by pytest-html 4.1+) embeds test data as JSON keyed by full pytest node ID in a `<div id="data-container">` element
- No cross-linking exists between the two reports

**Implementation approach:**
- In `tck/reporting/html_formatter.py`, change the test ID `<span>` elements into `<a>` links pointing to `tck_report.html` with an appropriate anchor or filter
- Investigate how pytest-html 4.1+ supports deep-linking to individual tests (e.g., URL hash fragments or query parameters) and use the appropriate mechanism
- If pytest-html doesn't support deep-linking natively, consider adding a custom anchor mechanism via a pytest-html hook or a post-processing step

**Key files:**
- `tck/reporting/html_formatter.py` — generates the compatibility HTML, renders test IDs
- `tck/reporting/aggregator.py` — aggregates test results with test_ids per requirement
- `tests/compatibility/conftest.py` — pytest hooks, session finalization
- `run_tck.py` — passes `--html` flag to pytest for tck_report.html generation

## Implementation Notes

<!-- SECTION:NOTES:BEGIN -->
pytest-html 4.1+ sets `id=testId` on each `<tbody>` element in tck_report.html, where `testId` is the full pytest node ID. This means we can use standard URL fragment linking (`tck_report.html#<node_id>`) to deep-link to individual tests.

The change was minimal — in `_format_test_ids()`, the `<span>` elements were converted to `<a>` links with `href="tck_report.html#<test_id>"`. The `cursor: help` CSS was removed since links have their own cursor.
<!-- SECTION:NOTES:END -->
